# Code of Conduct

## Our Standard

We want `awo` to be a respectful, technically serious, and welcoming project.

People participating in this project are expected to:

- be respectful and constructive
- assume good intent while giving clear feedback
- focus criticism on ideas, code, and behavior rather than people
- help keep the project accessible to newcomers
- avoid harassment, abuse, discrimination, and intimidation

## Unacceptable Behavior

Examples of unacceptable behavior include:

- personal attacks or demeaning language
- harassment, threats, or stalking
- discriminatory language or conduct
- publishing someone else's private information without permission
- deliberate disruption of discussions, reviews, or contributor workflows

## Scope

This code of conduct applies in project spaces, including issues, pull requests, discussions, reviews, and other public collaboration around the repository.

## Enforcement

Project maintainers may remove, edit, or reject contributions, comments, issues, or other interactions that violate this code of conduct.

Possible responses include:

- requesting a change in behavior
- moderating or removing content
- temporarily limiting participation
- banning repeat or severe offenders from project spaces

## Reporting

If you experience or witness unacceptable behavior:

- use GitHub’s moderation/reporting tools where appropriate
- contact the maintainers through the project’s available GitHub channels
- for security-sensitive concerns, follow [`SECURITY.md`](SECURITY.md) instead of posting exploit details publicly

Reports will be handled as carefully and respectfully as possible.
